## Key Interactions to Verify
- `run_portfolio(portfolio, prices)` returns correct NAV on a known price series
- `run_comparison(configs, prices)` partial failure: one bad config → results + errors, no raise
- Zero-crossing atomic rule: weight 0.5 → -0.3 → exactly 2 Trade records in same bar
- `max_leverage` enforcement: sum(abs(weights_t)) never exceeds Portfolio.max_leverage
- `detect_regime()` labels breadth series correctly with known thresholds
- `EnsembleStrategy` weights = weighted average of N sub-strategy outputs
- `assert_no_leakage()` fails when strategy uses shifted/future prices

## Edge Cases
- `analyze_cross_section()` with empty universe → ValueError
- `analyze_performance()` with 0 trades → all metrics = 0 (no divide-by-zero crash)
- `run_comparison([])` → ValueError("no configs provided")
- Factor compute on series with price=0 → FactorComputeError
- Weight matrix containing NaN → StrategyOutputError (never silently passes through)
- `EnsembleStrategy` with one failing sub-strategy → partial result + error logged

## Critical Paths
1. Full backtest: `DataLoader.load()` → `Factor.compute()` → `Strategy.compute()` → `run_portfolio()` → `analyze_performance()` — must work end-to-end from Jupyter with no Streamlit imports
2. Comparison sweep: `run_comparison(50 configs, prices)` → `ComparisonReport` with all 50 results ranked by Sharpe
3. Regime-filtered backtest: `Portfolio(regime_config=RegimeConfig(...))` → strategy receives RegimeSeries → trades reflect regime filter
4. No-leakage gate: `assert_no_leakage(output, prices)` catches a strategy that accidentally uses tomorrow's price

## Shared Test Fixtures (conftest.py)
- `synthetic_price_frame(symbol, days=500)` — deterministic OHLCV data, no network calls
- `synthetic_prices_dict(symbols, days=500)` — dict[str, PriceFrame] for multi-symbol tests
- `simple_strategy()` — deterministic weights (long signal every 20 days) for predictable output
- `known_trades()` — pre-built Trade list with known return_pct for performance metric validation
