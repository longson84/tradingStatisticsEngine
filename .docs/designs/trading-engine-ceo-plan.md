## Vision

### 10x Check
The same library that runs backtests also runs strategies live — consuming real-time price
feeds and submitting orders to a broker (Alpaca, Binance, IBKR) via a thin adapter.
The backtest and live paths share 100% of the Strategy code. The React dashboard shows
live equity curves updating via WebSocket. Paper trading first, real money when ready.

### Platonic Ideal
You open the dashboard. Five strategies are running simultaneously — three in backtest
mode, two in paper trading. The equity curve updates live. You click a trade and see
*why* it fired: which factors were active, what the breadth signal was, what regime the
market was in. You flip to Jupyter, `from trading_engine import run_backtest` — same
library, instant results. You write a new 20-line strategy, sweep 50 symbols, find the
optimal parameters, and click "Paper Trade" directly from the sweep result.

## Accepted Scope (added to this plan)

- **Regime detection:** `detect_regime(breadth_series, thresholds) → RegimeSeries` — Layer 3
  pure function. Labels each period as 'risk_on' | 'risk_off' | 'transition'.
- **Trade explainability:** Optional `ExplainRecord` on `Trade` dataclass. Fields:
  `active_factors: dict[str, float]`, `breadth_at_entry: float`, `regime_at_entry: str`.
  Populated when strategies pass factor context; None otherwise.
- **Strategy combiner:** Two meta-strategies implementing the Strategy Protocol:
  - `EnsembleStrategy(strategies, weights)` — weighted average of weight outputs
  - `RotationStrategy(strategies, selector)` — picks best-performing strategy per period
- **Comparison framework (OPEN QUESTION):** Batch/sweep/multi-strategy are all the same
  shape: run N configs, collect N PortfolioResults, rank them. Design a unified
  `run_comparison(configs) → ComparisonReport` before building. Needs design work first.

## Deferred to TODOS.md

## Open Questions

- **Comparison framework design:** What is the right API for `run_comparison`?
  - `run_comparison(strategy, symbols_list)` → batch
  - `run_comparison(strategy_class, param_grid, symbols)` → sweep
  - `run_comparison(strategies_list, symbols)` → multi-strategy
  All return `ComparisonReport`. The config objects vary; the runner is the same.
  Decision must be made before building Layer 4-5.
